SQL 基础（九）表和约束

1. 数据库设计

1.1 ER 图转数据库表

数据建模完成之后，可以把 E-R 图转换成数据库中的表：

  • 实体的名字转换为表。
  • 实体的属性转换为表中的列。
  • 具有唯一特点的属性设置为表中的主键。
  • 根据实体之间的关系，设置表中某列为外键列（主外键关联）。
    • 一对一关系：外键设可以设在任意一方。
    • 一对多关系：外键设在多的一方。
    • 多对多关系：需要建一张桥表。

1.2 主键（primary key）

表中经常有一个列或多列的组合，其值能唯一地标识表中的每一行。这样的一列或多列称为表的主键，通过它可强制表的实体完整性。

关系型数据库要求所有表都应该有主键，不过 Oracle 没有遵循此范例要求。Oracle 中的表可以没有主键（这种情况不多见）。

  • 作用：用来唯一标识一行数据的。
  • 特点：
    • 非空且唯一，相当于 not null + unique。
    • 主键所在列必须具有索引（主键的唯一约束通过索引来实现）。如果不存在，将会在索引添加的时候自动创建。

在表中可以使用一个有意义的列做主键。例如：学生表的学号、公民的身份证号。也可以使用没有意义的列做主键，因为主键的作用就是标识一行数据，大部分情况下都是用没有意义的列去做主键。此外，还能把多个列组合在一起作为表的主键。


1.3 外键（foreign key）

外键是用于建立和加强两个表数据之间的链接的一列或多列。通过将保存表中主键值的一列或多列添加到另一个表中，可创建两个表之间的链接。这个列就成为第二个表的外键。

  • 作用：连接两张表，体现两张表之间的对应关系。
  • 特点：
    • 表中的某一个声明为外键的列，一般是与之连接的表的主键（其实只要是具体唯一约束的列，就可以被另一种表的外键列所引用）。
    • 声明为外键的列和被引用的列，名字可以不同，但数据类型必须相同。
    • 如果外键列没有做主键或联合主键，那么外键列的值可以为空。
    • 实际上，外键列和被引用的列可以在同一张表中。最常见的就是表示一棵树。
    • 如果把 B 表中的联合主键，引用到 A 表中做外键，那么这个外键就是一个联合外键。

2. 约束

2.1 概念

列的约束，就是对这个列中的值的要求（限制）。

数据库中的约束有：

  • 主键约束 primary key
  • 外键约束 foreign key
  • 唯一约束 unique
  • 非空约束 not null （不能作为表级约束）
  • check 约束 check

按照约束位置的不同又可以分为：列级约束和表级约束。

如果要声明的约束为联合主键、联合外键、联合唯一的时候，一定要用表级约束。


2.2 constraint 关键字

在创建约束的时候，可以使用 constraint 关键字给约束命名。

例：创建一个课程表和教师表。


2.3 操作约束

2.3.1 增加约束

如果增加的是除 not null 以外的约束，需要使用 alter table 的 add 子句。

如果增加 not null 约束，那么必须使用 alter 语句的 modify 子句。


2.3.2 修改约束名

在同一个方案中，约束名必须惟一，并且约束名也不能与其他对象同名。


2.3.3 删除约束

当删除特定表的主键约束时，如果该表具有相关的从表，那么在删除主键约束时必须带有 cascade 选项。


2.3.4 禁用约束

禁止约束指使约束临时失效。当禁止了约束之后，约束规则将不再生效。在使用 sql * loader 或 insert 装载数据之前，为了加快数据装载速度，应该首先禁止约束，然后装载数据。

2.4 约束的状态

2.4.1 状态类型

很多时候由于业务需要，比如我们有大量的历史数据，需要和现有数据合并。当前表存在数据库约束（如非空约束），而这些历史数据又包含违背非空约束的数据行，为了避免导入时由于违反约束而导入失败，我们通过调整约束状态来达到目的。

数据库约束有两类状态：

  • 启用/禁用（enable/disable）：是否对新变更的数据启用约束验证。

  • 验证/非验证 (validate/novalidate) ：是否对表中已客观存在的数据进行约束验证。


这两类四种状态从语法角度讲可以随意组合，默认是 enable 和 validate。
状态 描述
enable validate 默认的约束组合状态，无法添加违反约束的数据行，数据表中也不能存在违反约束的数据行。
enable novalidate 无法添加违反约束的数据行，但对已存在的违反约束的数据行不做验证。
disable validate 可以添加违反约束的数据行，但对已存在的违反约束的数据行会做约束验证。从描述中可以看出来，这本来就是一种相互矛盾的约束组合，只不过是语法上支持这种组合罢了，造成的结果就是会导致 DML 失败。
disable novalidate 可以添加违法约束的数据行，对已存在的违反约束的数据行也不做验证。

3. 表

2.1 命名规则

表和列的命名规则：

  • 名字中只能出现字母、数字、下划线（_）和井号（#）。
  • 必须是字母开头。
  • 不能是数据库中的关键字。
  • 长度必须在 1 ~ 30 个字符之间。
  • 不能和数据库中已有的对象重名。

2.2 数据类型

Oracle 数据库中常用的几种数据类型：

数据类型 描述
char(size) 定长的字符串。size 用于指定字符串的最大长度，size 的值必须是正整数且不超过 32767。字符串长度没有达到 size 用空格填充。
varchar(size) 变长的字符串。size 用于指定字符串的最大长度，字符串长度没有达到 size 不会填充。可以存空字符串和 null。
varchar2(size) 变长的字符串。varchar2 是 Oracle 数据库特有的类型。用法和 varchar 一样，但是不能存储空字符串。
number(p, s) 数字。p 表示最大位数（整数位 + 小数位）。s 表示保留的小数位（四舍五入），也可以为负数。可以直接使用 number，不加参数。
date 日期类型。
blob 存二进制对象，例如视频，音频，图片等。
clob 存储大文本，例如很多很多文字。
