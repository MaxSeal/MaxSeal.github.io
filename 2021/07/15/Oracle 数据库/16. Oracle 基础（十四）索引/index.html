1. 概念

索引是 Oracle 数据库中一种与表关联的可选对象。索引类似于书的目录，使用索引能有效提高 SQL 语句的查询速度。

Oracle 会自动地维护索引。一旦建立了索引，用户插入、更新和删除数据，Oracle 会自动更新索引。


2. 原理

因为 DB 在执行一条 SQL 语句的时候，默认的方式是根据搜索条件进行全表扫描，遇到匹配条件就加入搜索结果集合。如果我们对某一字段增加索引，查询时就会先去索引列表中一次定位到特定值的行数，大大减少遍历匹配的行数，所以能明显增加查询的速度。

创建了索引并不一定就会使用，因为 Oracle 在自动统计表的信息后，会决定是否使用索引， 表中数据很少时，使用全表扫描速度已经很快了，那么就没有必要使用索引了。


3. 创建和删除

3.1 创建

创建索引有两种情况：

  • 自动创建索引：

    当在表中指定了 primary key 或者 unique 约束时，会自动创建唯一值索引。

  • 用户创建索引

    用户可以创建非唯一值索引以提高在访问数据时的效率。

在某一个列上加入上了索引，那么也只有在数据量很大的时候，才能有所体现出这个查询的效率。并且索引一旦建立成功就由数据库来管理，我们自己是控制不了的。

索引不是越多越好。过多的索引反而会降低查询效率。


3.2 创建索引的原则

应该创建索引的列：

  • 经常作为 where 子句的限定条件或连接条件的列，适合创建索引。

  • 包含大量非空数据的列，适合创建索引。

  • 经常需要 group by，order by ，distincet 的列，适合创建索引。

  • 经常需要查询的列。


不应该创建索引的列：

  • 很少被查询的列。
  • 经常需要修改的列。
  • 数据很少的列。
  • 数据类型为 blob 的列。
