1. 概述

java.util.stream.Stream 接口，表示能应用在一组元素上，一次执行的操作序列，也就是可以对一组数据进行连续的多次操作。

Stream 在使用的时候，需要指定一个数据源，比如 java.util.Collection 的子类， List 或者 Set 都可以，但是 Map 类型的集合不支持。

Stream 是对集合功能的增强，它提供了各种非常便利、高效的聚合操作，可以大批量数据操作，同时再 结合Lambda表达式，就可以极大的提高编程效率。

Stream 的 API 提供了串行和并行两种模式进行操作数据。

2.2 数组

只要是 Collection 类型的集合，都可以调用 stream 方法，将集合转换为 Stream 对象。


stream 方法是 Collection 接口里的一个默认方法。所以只要是 Collection 类型的集合，就可以调用 stream 方法。


2.4 基本类型

对于基本数值类型，有专门的三种Stream类型：

  • IntStream
  • LongStream
  • DoubleStream

虽然也可以用Stream类型，并指定泛型。但是，在数据量较大的时候，自动拆箱/装箱会比较消耗性能，所以提供了上面三种专门针对基本类型的 Stream。

4. Stream 操作

Stream操作分为中间操作或者最终操作两种：

  • 中间操作，返回Stream本身，这样就可以将多个操作依次串起来。
  • 最终操作，返回一特定类型的计算结果。

4.1 中间操作

常用的中间操作，有以下几种：

4.1.1 filter

过滤方法，返回满足 predicate 指定的条件的所有元素的一个新流。


4.1.2 map

对调用流中的元素，应用 Function 所指定的操作，然后返回一个新流。


4.1.3 reduce

reduce 方法是将一组数据俩俩合并，最后得出一个结果。


4.1.4 sorted

将流里的数据进行排序。


4.1.5 limit

返回 Stream 的前面 n 个元素。


4.1.6 skip

跳过前 n 个元素只要后面的元素。


4.1.7 distinct

去除重复数据。


4.1.8 Stream.concat

拼接两个流。


4.1.9 Stream.generate

通过 Supplier 接口,可以自己来控制数据的生成。


4.1.10 Stream.iterate

它跟 reduce 操作很像，需要接受一个起始值（种子），然后通过函数得出一个 结果，再把结果当做参数传给函数，再得出第二个结果，依次类推，其实就是一个递归操作。

假设起始值为seed，函数为f，那么第一个元素是seed，第二个元素是f(seed)，第三个元素是f(f(seed)) ， 第四个元素是f(f(f(seed)))，以此类推。


4.2 最终操作

4.2.1 iterator

返回迭代器对象。


4.2.2 forEach

将调 Stream 中的每个元素，交给一个 Consumer 函数处理。


4.2.3 count

统计流中的元素数，并返回结果。


4.2.4 max & min

返回流中基于 comparator 所指定的比较规则，比较出的最大值/最小值。


4.2.5 toArray

使用调用流中的元素，生成数组返回。


4.2.6 collect

将元素收集到一个可以修改的容器中，并返回该容器。


4.2.7 xxxMatch

匹配操作，Stream 中提供了多种匹配模式。


4.28 findFirst

返回 Stream 的第一个元素。


5. 并行流

Stream 有串行和并行两种。串行 Stream 上的操作是在一个线程中依次完成，而并行 Stream 则是在多个线程上同时执行。

创建并行Stream的俩种方式：

  • 调用串行 Stream 的 parallel() 方法，可以将其转换并行 Stream。


  • 调用集合对象的 parallelStream 方法，之后获取并行 Stream。
