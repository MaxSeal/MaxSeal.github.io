SQL 基础（三）单行函数和转换函数

单行函数也被称为单值函数，每操作一行数据（某个字段值）都会返回结果。

转换函数可以将一个类型的数据，转换为另一种类型的数据。

1. 哑表

Oracle 数据库中有一张特殊的表：dual（哑表）。

dual 之所以被称为哑表，是因为它是一个 Oracle 内部自动创建的单行单列的虚拟表。这个表中只有一个列 dummy ，其数据类型为 verchar2(1)。Oracle 右内部逻辑保证 dual 表中永远只有一条数据，且只有 Oracle 数据库中有哑表。


在实际的使用中，dual 表主要是用来选择系统变量或求一个表达式的值。

例如：求表达式 1+1 的结果。

按照 SQL 语句的要求，没有表就没法查询，而表达式 1+1 不属于任何表，那么可以使用哑表。


2. 单行函数

单行函数也被称为单值函数，每操作一行数据（某个字段值）都会返回员工结果。
单值函数可以分为：字符函数、日期函数和数值函数。

2.1 字符函数

常用的字符函数有：

函数 作用
lower(x) 将字符串 x 转为小写。
upper(x) 将字符串 x 转为大写。
initcap(x) 将字符串 x 的首字母转为大写。
length(x) 获取字符串 x 的长度。
ascll(x) 返回字符串的 ASCII 码。
concat(x, y) 连接两个字符串 x 和 y。
replace(x, old_str, new_str) 在字符串 x 中用 new_str 替换 old_str。
lpad(x, width [,pad_str]) 指定字符串 x 的长度。不够长度时，在左边填充。
rpad(x, width [,pad_str]) 指定字符串 x 的长度。不够长度时，在右边填充。
trim([trim_str from] x) 把 x 的两边截去 trim_str 字符串，缺省截去空格。
ltrim(x [,trim_str]) 把 x 的左边截取 trim_str 字符串，缺省截去空格。
rtrim(x [,trim_str]) 把 x 的右边截取 trim_str 字符串，缺省截去空格。
instr(x, str [, start] [,n]) 从 x 中查找 str。
可以指定从 start 开始，也可以指定从 n 开始。
substr(x, start [,length]) 在字符串 x 上截取一个子串。
start 是开始位置，length 是截取的长度。
缺省 length ，默认到结尾。

2.1.1 lower

lower(x)：将字符串 x 转为小写。


2.1.2 upper

upper(x)：将字符串 x 转为大写。


2.1.3 initcap

initcap(x)：将字符串 x 的首字母转为大写，其他字母小写。


2.1.4 length

length(x)：获取字符串 x 的长度。


2.1.5 ascii

ascii(x)：获取字符 x 的 ascii 码。


2.1.6 concat

concat(x, y)：连接连个字符串 x 和 y。


2.1.7 replace

replace(x, old_str, new_str)：在字符串 x 中用 new_str 替代 old_str。

情况 1：字符串 x 中不存在 old_str。则执行后的结果是 x。


情况 2：不写 new_str，执行结果为在 x 中去掉了 old_str。


情况 3：正常情况


2.1.8 lpad 和 rpad

lpad(x, width [,pad_str])：指定字符串 x 的长度。不够长度时，如果没有指定填充字符，则使用空格在左边填充。如果指定了填充字符就使用填充字符在左边填充。


2.1.9 trim

trim([trim_str from] x)：把 x 的两边截去 trim_str 字符（1个），缺省截去空格。

情况 1：不指定 trim_str。截去 x 两边的所有空格。


情况 2：trim_str 为空字符串，执行的结果也为空。


情况 3：trim_str 不为空串，但是 x 两边没有 trim_str。如果这时 x 两边有空格，则会两边各去掉一个空格。


情况 4：trim_str 不为空串，但是 x 只有一边含有 trim_str。如果这时 x 的另外边有空格，则会在这边去掉一个空格。


情况 5：trim_str 不为空串，但是不是一个字符。


情况 6：正常情况。


2.1.10 ltrim 和 rtrim

ltrim(x [,trim_str])：把 x 的左边截取 trim_str 字符串，缺省截去空格。

情况 1：trim_str 的长度比 x 长。执行结果为 null。


情况 2：trim_str 为空。执行结果为 null。


情况 3：trim_str 与 x 相同。执行结果为 null。


情况 4：x 和 trim_str 从左边第一个字符开始一一比较，没有相同的部分。执行结果就是 x。但如果 x 的左边含有空格，则会截去一个空格。


情况 5：正常情况。

x 和 trim_str 从左边第一个字符开始一一比较，只会截去相同的部分。


2.1.11 instr

instr(x, str [, start] [,n])：从 x 中查找 str。可以指定从 start 开始，也可以指定从 n 开始。


2.1.12 substr

substr(x, start [,length])：在字符串 x 上截取一个子串。start 是开始位置，length 是截取的长度。缺省 length ，默认到结尾。


2.2 数值函数

专门操作数字的函数，常用的数字函数有：

函数 作用
abs(x) x 的绝对值。
cos(x) x 的反余弦。
acos(x) x 的余弦。
sin(x) x 的正弦。
asin(x) x 的反正弦。
ceil(x) 大于或等于X的最小值(整数)。
floor(x) 小于或等于X的最大值(整数)。
round(x [,y]) x 在第 y 位四舍五入。
trunc(x [,y]) x 在第 y 位截断。
power(x,y) x 的 y 次幂。
sqrt(x) x 的平方根。
log(x,y) x 为底 y 的对数。
mod(x,y) x 除以 y 的余数。

2.2.1 ceil

ceil(x)：大于或等于 X 的最小值(整数)。


2.2.2 floor

floor(x)：小于或等于 X 的最小值(整数)。

2.2.3 round

round(x [, y])：x 在 y 为四舍五入。

  • x 表示要进行四舍五入操作的数字。
  • y 表示要保留到哪一位，默认是 0 ，即保留到个位。

2.2.4 trunc

trunc(x [,y])：x 在第 y 位截断。

trunc 的用法和 round 基本一样，但是 trunc 只会舍去不会进位。


2.2.5 mod

mod(x, y)：x 除以 y 的余数。


2.3 日期函数

2.3.1 sysdate

sysdate 是 Oracle 中用来表示当前时间的关键字，并且可以使用它来参与时间运算。 sysdate 参与时间的加减操作的时候，单位是天。


2.3.2 常用的日期函数

函数 作用
months_between(x, y) 求两个日期之间相差的月数。
add_months(x, y) 返回一个日期数据：表示一个时间点，往后推 x 月的日期。
next_day(x, 星期 y) 返回一个日期数据：表示一个时间点后的下一个星期几在哪一天。
last_day(x) 返回一个日期数据：表示一个日期所在月份的最后一天。
round(x, y) 对日期进四舍五入，返回操作后的日期数据。
trunc(x, y) 对日期进行截取和 round 类似，但是只舍弃不进位。

2.3.3 months_between

months_between(x, y)：求两个日期之间相差的月数。


2.3.4 add_months

add_months(x, y)：返回一个日期数据：表示一个时间点，往后推 x 月的日期。 y 也可以是负数，表示之前多少月


2.3.5 next_day

next_day(x, 星期)：返回一个日期数据：表示一个时间点后的下一个星期几在哪一天。


2.3.6 last_day

last_day(x)：返回一个日期数据：表示一个日期所在月份的最后一天。


2.3.7 round

round(x, y)：对日期进四舍五入，返回操作后的日期数据。


2.3.8 trunc

trunc(x, y)：对日期进行截取和 round 类似，但是只舍弃不进位。


3. 转换函数

转换函数可以将一个类型的数据，转换为另一种类型的数据。

转换函数主要有三种：

  • to_char 把一个数字或日期数据转换为字符。
  • to_number 把字符转换为数字。
  • to_date 把字符转换为日期。

3.1 to_char

3.1.1 数字转换为字符

数字转为字符的常用格式：

格式 示例 说明
9 999 指定位置处显示数字。
. 9.9 指定位置显示小数点。
， 99,999 指定位置显示逗号。
$ $999 数字开头显示一个美元符号。
L L999 数字开头显示一个本地的货币符号。
PR 999PR 如果数字式负数则用尖括号进行表示。

3.1.2 日期转换为字符

日期转换为字符的常用格式：

格式 说明
yyyy 四位数的年份。
rrrr 四位数的年份。
yy 两位数的年份。
rr 两位数的年份。
mm 两位数的月份（数据）。
D 一周的星期几。
DD 一个月的第几天。
DDD 一年的第几天。
YEAR 英文的年份。
MONTH 英文的全称月份。
mon 英文的简称月份。
ddsp 英文的第几天（一个月的）。
ddspth 英文序列数的第几天（一个月的）。
DAY 英文的全称星期（monday, tuesday…）
DY 英文的简称星期（mon, tues…）
hh 小时
mi 分钟
ss 秒

3.2 to_number

to_number(字符)：把字符转换为数值。


3.3 to_date

to_date(字符串, 日期格式)：把字符转换为日期。


3.4 千年虫问题

在早期的计算机的程序中，规定了的年份仅用两位数来表示。 假如是 1971 年，在计算机里就会被表示为 71。 但是到了 2000 年的时候这个情况就出现了问题，计算机读取出之前存的一个日期88，结果就是 2088 年。 这样的话，计算机内部对年份的计算就会出现问题，这个事情当时被称为千年虫。

为了处理这个问题，后面就有了一种新的日期格式：rr 。
如果在存储日期数据的时候，年份采用的格式为 rr，并且只提供了最后 2 位年份，那么年份中的前两位数字就由两部分共同确定，其规则如下：

  • 规则 1：
    如果指定年在 00 ~ 49 之间，并且当前年份在 00 ~ 49 之间，那么指定年的世纪就与当前年份的世纪相同。因此，指定年的前两位数字就等于当前年份的前两位数字。例如：如果指定年为15，而当前年 份为 2007，那么指定年就是 2015。

  • 规则 2：
    如果指定年在 50 ~ 99 之间，并且当前年份在 00 ~ 49 之间，那么指定年的世纪就等于当前年份的世纪减去1。因此，指定年的前两位数字等于当前年份的前两位数字减去 1。例如：如果指定年为 75，而当 前年份为 2007，那么指定年就是 1975 。

  • 规则 3：
    如果指定年在 00 ~ 49 之间，并且当前年份在 50 ~ 99 之间，那么指定年的世纪就等于当前年份的世纪加 上1。因此，指定年的前两位数字等于当前年份的前两位数字加上 1。例如：如果指定年为 15，而当 前年份为 2075，那么指定年就是 2115。

  • 规则 4：
    如果指定年在 50 ~ 99 之间，并且当前年份在 50~99 之间，那么指定年的世纪就与当前年份的世纪相同。因此，指定年的前两位数字就等于当前年份的前两位数字。例如：如果指定年为 55，而当前年 份为 2075，那么指定年就是 2055。
