Oracle 数据库结构

数据库服务器是信息管理的关键。

通常，数据库服务器能在多用户的环境下地靠地管理大量数据，以便用户可以并发地访问相同的数据。此外，数据库服务器还可以防止未经授权的访问，并为故障恢复提供有效的解决方案。


1. 数据库和实例

Oracle 数据库服务器由一个数据库和至少一个数据库实例组成，通常被简称为实例。

因为数据库和实例联系紧密，所以 Oracle 数据库这个术语有时同时用指实例和数据库。但从严格的意义上来讲，这些术语是有区别的：

  • 数据库

    数据库是一组位于磁盘上的用于存储数据文件。这些文件能够独立于数据库实例存在。

  • 数据库实例

    数据库实例是一组管理数据库文件的内存结构。数据库实例由一个被称为 system global area (SGA) 的共享存储区域和一些列的后台进程组成。实例可以独立于数据库文件而存在。

对于到实例的每个用户连接，一个客户端进程运行应用程序。每个客户端进程都与其自己的服务器进程相关联。服务器进程有自己的私有会话内存，称为 program global area (PGA)。


2. 数据库的存储结构

可以从物理和逻辑的角度来考虑 Oracle 数据库的结构。

数据库拥有物理结构和逻辑结构。因为物理结构和逻辑结构是分开的，所以修改数据的物理存储结构并不会影响其逻辑存储结构。


2.1 物理存储结构

物理结构是由构成数据库的操作系统文件所决定，Oracle 数据库文件包括：

  • 数据文件（Data Files）：

    每个 Oracle 数据库都有一个或多个物理数据文件，其中包含了所有的数据库数据。数据库逻辑结构（如表和索引）的数据物理地存储在数据文件中。名字的格式通常为*.dbf格式，例如: userCIMS.dbf 。


  • 日志文件（Redo Log File）：

    日志文件用于记录数据库所做的全部变更（如增加、删除、修改），以便在系统发生故障时，用它对数据库进行恢复。名字通常为Log*.dbf格式，如：Log1CIMS.dbf,Log2CIMS.dbf


  • 控制文件（Control File）:

    每个 Oracle 数据库都有一个控制文件，它们是较小的二进制文件，用于记录数据库的物理结构，如：数据库名、数据库的数据文件和日志文件的名字和位置等信息。用于打开、存取数据库。名字通常为 Ctrl*.ctl 格式，如 Ctrl1CIMS.ctl。　　


  • 配置文件：

    配置文件用于记录 Oracle 数据库运行时的一些重要参数，如：数据块的大小，内存结构的配置等。名字通常为 init*.ora 格式，如：initCIMS.ora 。
