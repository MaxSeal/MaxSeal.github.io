abstract 关键字的使用

abstract 方法和类的使用方式和注意事项。

1. abstract 关键字

abstract修饰符，可以修饰类、方法。

1.1 abstract 方法

  • 抽象方法是一种没有任何实现的方法，该方法的的具体实现由子类提供。

  • 抽象方法不能被声明成 final 和 static。

  • 任何继承抽象类的子类必须实现父类的所有抽象方法，除非该子类也是抽象类。

  • 如果一个类包含若干个抽象方法，那么该类必须声明为抽象类。抽象类可以不包含抽象方法。

  • 抽象方法的声明以分号结尾。

1. 2 abstract 类

  • 抽象类不能用来实例化对象，声明抽象类的唯一目的是为了将来对该类进行扩充。

  • 一个类不能同时被 abstract 和 final 修饰。

  • 如果一个类包含抽象方法，那么该类一定要声明为抽象类，否则将出现编译错误。

  • 抽象类可以包含抽象方法和非抽象方法。

2. 接口

  • 接口的内部主要就是封装了抽象方法和静态常量。

  • 接口的定义和类很类似，但是接口需要使用 interface 关键字来定义。

  • 接口最终也会被编译成.class文件。

  • 接口里面的属性，默认就是 public static final 修饰的。

  • 接口里面的方法，默认就是 public abstract 修饰的。

  • JDK8中，还允许在接口中编写静态方法和默认方法。

  • JDK9中，还允许在接口中编写私有方法。

2.1 接口实现
  • 类和类之间的关系是继承，类和接口之间的关系是实现。
  • 一个类只能继承一个类，但是能实现了一个或多个接口。
  • 一个类实现了接口，那么就要实现接口中所有的抽象方法，否则这个类自己就必须声明为抽象类。
2.2 接口继承

类和类之间是单继承，接口和接口之间是多继承。

2.3 接口的引用指向它的实现类对象

多态的前提是继承，必须要先有子父类关系才行，而类和接口之间的实现关系，其实也是继承的一 种形式，所以在类和接口的实现关系中，也可以使用多态。
