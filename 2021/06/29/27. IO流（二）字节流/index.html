IO 流（二）字节流

常见的字节流及其用法。

1. 字节流

计算机中一切数据（文本、图片、视频等）都是以二进制的形式存储的，最终都是一个一个的字节。所以使用流，进行传输数据的时候，也是一个一个的字节进行的。

一般情况，使用字节流来操作数据的时候，往往是使用一对一。一个字节输入流，负责读取数据；一 个字节输出流，负责将数据写出去，而这些流都将是 InputStream 和 OutputStream 的子类型。


在代码中，使用流操作数据的的基本步骤是： 1. 声明流 2. 创建流 3. 使用流 4. 关闭流。

3. 字节数组

3.1 ByteArrayInputStream

3.1.1 属性

在 ByteArrayInputStream 类中有 4 个属性。

属性 说明
buf[] 由流的创建者提供的字节数组。元素 buf[0] 到buf [count-1] 是唯一可以从流中读取的字节；元素 buf[pos] 是要读取的下一个字节。
pos 要从输入流缓冲区读取的下一个字符的索引。 此值应始终为非负且不大于 count 的值。 要从输入流缓冲区读取的下一个字节将是 buf[pos] 。
mark 流中当前标记的位置。 ByteArrayInputStream 对象在构造时默认标记在位置零。
它们可以通过 mark() 方法在缓冲区内的另一个位置进行 mark() 。 当前缓冲区位置由 reset() 方法设置到这一点。
如果未设置标记，则标记的值是传递给构造函数的偏移量。（如果未提供偏移量，则为 0）
count 比输入流缓冲区中最后一个有效字符大一的索引。 此值应始终为非负且不大于 buf 的长度。 它比 buf 中可以从输入流缓冲区读取的最后一个字节的位置大一个。

3.1.2 构造器

在 ByteArrayInputStream 类中有 2 个构造器。

3.1.3 常用方法

**int read()**：从此输入流中读取下一个数据字节。

  • 值字节以 0 到 255 范围内的 int 形式返回。
  • 如果由于已到达流末尾而没有可用字节，则返回值 -1 。
  • 这种read方法不能阻塞。

**int read(byte b[], int off, int len)**：从此输入流中将最多 len 个字节的数据读入字节数组。

  • 如果 pos 等于 count ，则返回 -1 以指示文件结束。
  • 读取的字节数 k 等于 len 和 count-pos 的较小者。 如果 k 为正数，则字节 buf[pos] 到 buf[pos+k-1] 以 System.arraycopy 执行的方式复制到 b[off] 到 b[off+k-1] 中。 将值 k 添加到 pos 并返回 k 。
  • 这种 read 方法不能阻塞。

**long skip(long n) **：从这个输入流跳过 n 个字节的输入。返回实际跳过的字节数。

  • 如果到达输入流的末尾，可能会跳过更少的字节。
  • 要跳过的实际字节数 k 等于 n 和 count-pos 的较小者。 将值 k 添加到 pos 并返回 k 。

**int available()**：返回可以从此输入流读取（或跳过）的剩余字节数。

  • 返回的值是 count - pos ，它是要从输入缓冲区读取的剩余字节数。
  • 可以在不阻塞的情况下从此输入流读取（或跳过）的剩余字节数。

3.2 ByteArrayOutputStream

3.2.1 属性

ByteArrayOutputStream 中只有 2 个属性。

属性 描述
buf[] 存储数据的缓冲区。
count 缓冲区中的有效字节数。

3.2.2 构造器

在 ByteArrayOutputStream 类中有 2 个构造器。

3.2.3 常用方法

**void write(int b)**：将指定的字节写入此 ByteArrayOutputStream 。

**void write(byte b[], int off, int len) **：将指定字节数组中的 len 个字节从偏移量 off 开始写入此 ByteArrayOutputStream。

**int size()**：返回缓冲区的当前大小，即属性 count 的值。

**void reset()**：将此 ByteArrayOutputStream 的 count 字段重置为零，以便丢弃输出流中当前累积的所有输出。 输出流可以再次使用，重用已经分配的缓冲区空间。

**void writeTo(OutputStream out)**：将此 ByteArrayOutputStream 的完整内容写入指定的输出流参数，就像使用 out.write(buf, 0, count)调用输出流的 write 方法一样。

**byte[] toByteArray()**：创建一个新分配的字节数组。 它的大小是此输出流的当前大小，并且缓冲区的有效内容已复制到其中。

**String toString()**：使用平台的默认字符集将缓冲区的内容转换为字符串解码字节。 新String的长度是字符集的函数，因此可能不等于缓冲区的大小。

4. 管道

使用字节流，可以从管道中读取数据，以及向管道中写数据。

  • java.io.PipedInputStream 负责从管道中读取数据。
  • java.io.PipedOutputStream 负责将数据写入到管道中。

一般可以在一个线程中，使用管道输出流，将数据写入到管道中，在另一个线程中，读取管道中的数据。


4.1 PipedInputStream

4.1.1 构造器

PipedInputStream 有 4 个构造器。

4.1.2 常用方法

connect(PipedOutputStream src)：连接一个 PipedOutputStream 对象。

int read()：从管道缓冲区中读取一个字节，并把这个值字节以 0 到 255 范围内的 in t形式返回。 此方法会阻塞，直到输入数据可用、检测到流结束或抛出异常为止。

int read(byte b[], int off, int len)：将管道缓冲区里的数据读入到 b[ ] 的指定部分，并返回实际读入字节的个数。如果由达到流的末尾，则返回 -1 。此方法会阻塞，直到至少有一个输入字符可用。

int available()：返回可以在不阻塞的情况下从此输入流读取的字节数。

public void close()：关闭流。
