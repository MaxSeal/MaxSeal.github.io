2.1.3 常用方法

public int read()：读入单个字符。如果到达流的末尾则返回 -1，否则返回字符对应的整数值。

public int read(char b[], int off, int len)：将流中的字符数据读入到 b[ ] 数组的一部分。如果到达流的末尾则返回 -1，否则返回实际读入的字符数量。

public long skip(long n)：跳过字符。 返回实际跳过的字符数。 如果n为负，则此方法不执行任何操作并返回0 。

public void mark(int readAheadLimit)：标记流中的当前位置。 对 reset() 的后续调用会将流重新定位到这一点。

public void reset()：将流重置为最近的标记，如果从未标记过，则重置为开头。

public void close()：关闭流。

3. 管道

使用字符流，可以从管道中读取数据，以及向管道中写数据。

  • java.io.PipedReader 负责从管道中读取数据。
  • java.io.PipedWriter 负责将数据写入到管道中。

一般可以在一个线程中，使用管道输出流，将数据写入到管道中，在另一个线程中，读取管道中的数据。


3.1 PipedReader

3.1.1 构造器


3.1.2 常用方法

public void connect(PipedWriter src)：连接一个 PipedWriter 对象。

public synchronized int read()：从管道缓冲区中读取一个字符，并返回这个字符的整数值。 如果由达到流的末尾，则返回 -1 。 此方法会阻塞，直到输入数据可用、检测到流结束或抛出异常为止。

public synchronized int read(char cbuf[], int off, int len)：将管道缓冲区里的数据读入到cbuf[ ] 的指定部分，并返回实际读入字符的个数。如果由达到流的末尾，则返回 -1 。此方法会阻塞，直到至少有一个输入字符可用。

public void close()：关闭流。


3.2 PipedWriter

3.2.1 构造器


3.2.2 常用方法

public synchronized void connect(PipedReader snk)：连接一个 PipedReader 对象。

public void write(int c)：将指定字符 c 写入到与之连接的 PipedReader 对象的管道缓冲区中。如果线程正在从连接的管道输入流中读取数据字符，但该线程不再处于活动状态，则会抛出IOException 。

public void write(char cbuf[], int off, int len)：将字符数组 cbuf[ ] 的指定部分写入到与之连接的 PipedReader 对象的管道缓冲区中。如果线程正在从连接的管道输入流中读取数据字符，但该线程不再处于活动状态，则会抛出 IOException 。

public synchronized void flush()：刷新此输出流并强制写出任何缓冲的输出字符。

public void close()：关闭流。


3.3 使用

  • PipedWriter 和 PipedReader 未进行连接都不能正常使用。
  • 先使用 PipedWriter 的 write 方法写入数据，才能使用 PipedReader 的 read 方法读数据。
  • PipedWriter 类内有一个私有属性 PipedReader sink。 当 PipedWriter 对象与 PipedReader 对象连接时，会将 PipedReader 对象的引用赋值给 sink。
  • PipedWriter 的 write 方法实际上是调用了与之连接的 PipedReader 对象的 receive 方法，将 write 方法写入的数据储存在该 PipedReader 对象的管道缓冲区中。
